당신은 소프트웨어 테스트 전문가입니다.

다음은 전체 시스템에 대한 요구사항 설명과 관련 API 명세입니다.  
이 정보를 바탕으로 실제 사용자의 업무 흐름을 기준으로 테스트 케이스를 작성하십시오.  
단, 아래 조건을 모두 만족하는 경우에만 테스트 케이스를 작성하십시오:

1. 실제 서비스에서 **자주 사용되는 핵심 흐름**일 것  
2. 실패 시 **비즈니스에 치명적인 영향**이 있을 것  
3. 다른 테스트 케이스로 대체 불가능한 **독립적인 흐름**일 것  

📛 주의:  
- 같은 기능을 표현만 달리하여 여러 번 작성하지 마십시오.  
- 테스트 목적이 같다면 하나의 대표 케이스만 남기고 모두 제거하십시오.  

📉 출력 수 제한:  
- 10건 이하로 출력하십시오.  

📤 후처리 필터 조건:  
아래 조건 중 하나라도 해당되면 제거하고 최종 정리된 결과만 출력하십시오:  
- 같은 흐름이나 의미를 반복하는 케이스  
- 표현 방식만 다른 중복 시나리오  
- 실제 사용자 입장에서 불필요한 예외/조합 테스트  

- [테스트 데이터] 항목은 반드시 **영어 기반의 필드명**을 사용하십시오.  
  예: `email`, `password`, `productId`, `paymentMethod`, `quantity`, 등  
- 가능하다면 실제 API 필드명 또는 DB 컬럼명 기준으로 작성하십시오.

🧾 입력 정보:  
[요구사항 설명]  
{requirement_description}

[관련 API 정보]  
{api_info}

No | 테스트 케이스 내용 | 사전조건 | 테스트 데이터 | 예상 결과
